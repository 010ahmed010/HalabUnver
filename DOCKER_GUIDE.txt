HalabUnver - Docker Deployment Guide
======================================

REQUIREMENTS
------------
- Docker installed on your VPS
- A MongoDB Atlas connection string (Atlas must whitelist your VPS IP or use 0.0.0.0/0)
- A strong JWT secret key (generate it once and save it safely)
- Nginx installed on the host VPS (handles SSL and forwards traffic to the container)


GENERATE YOUR JWT SECRET (do this once)
-----------------------------------------
Run this command on any machine that has Node.js:

    node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

Copy the output and save it somewhere safe (password manager, etc.).
You MUST use this exact same value every time you run or restart the container.


BUILD THE DOCKER IMAGE
-----------------------
Upload your project files to the VPS, then from the project root run:

    docker build -t halabunver:v1.0 .

This builds both the React frontend and the Node.js backend in one image.
The React app is compiled first, then its output is bundled inside the backend image.

To rebuild after code changes:

    docker build --no-cache -t halabunver:v1.0 .


RECOMMENDED: USE AN ENV FILE (more secure than inline flags)
-------------------------------------------------------------
Create a file at /etc/halabunver.env on your server (readable only by root):

    MONGODB_URI=your_mongodb_atlas_connection_string
    JWT_SECRET=your_saved_jwt_secret
    JWT_EXPIRES_IN=7d
    NODE_ENV=production
    PORT=8000

Secure the file:

    chmod 600 /etc/halabunver.env


RUN THE CONTAINER
------------------
Using env file (recommended):

    docker run -d \
      --restart unless-stopped \
      --env-file /etc/halabunver.env \
      -p 8000:8000 \
      --name halabunver \
      halabunver:v1.0

Using inline flags (quick testing only — secrets are visible in process list):

    docker run -d \
      --restart unless-stopped \
      -p 8000:8000 \
      -e MONGODB_URI=your_mongodb_connection_string \
      -e JWT_SECRET=your_saved_jwt_secret \
      -e JWT_EXPIRES_IN=7d \
      -e NODE_ENV=production \
      --name halabunver \
      halabunver:v1.0

NOTE: --restart unless-stopped means the container automatically restarts
if it crashes or if the server reboots. Always include this flag in production.

NOTE: The container runs on port 8000 internally. Nginx on the host listens
on port 80/443 and forwards traffic to it. Do not expose port 80 directly
from Docker — let host Nginx handle it.


CONFIGURE HOST NGINX
---------------------
Create a new site config on your VPS:

    sudo nano /etc/nginx/sites-available/halabunver

Paste this (replace yourdomain.com with your actual domain):

    server {
        listen 80;
        server_name yourdomain.com www.yourdomain.com;

        location / {
            proxy_pass http://localhost:8000;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }

Enable it and reload Nginx:

    sudo ln -s /etc/nginx/sites-available/halabunver /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx


ENABLE SSL (HTTPS)
-------------------
Install Certbot if not already installed:

    sudo apt install -y certbot python3-certbot-nginx

Run Certbot — it auto-updates your Nginx config with SSL:

    sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

SSL certificates renew automatically. Done.


IMPORTANT - JWT SECRET RULE
-----------------------------
The JWT secret is used to sign and verify user login tokens.

- Every time you restart or redeploy the container, you MUST use
  the exact same JWT_SECRET value.

- If you use a different secret after a restart, all currently logged-in
  users will be logged out and must log in again.

- If you do not pass JWT_SECRET, the server will refuse to start
  and print a FATAL error. This is intentional for security.


IMPORTANT - MONGODB CONNECTION
--------------------------------
- In development (Replit), if MONGODB_URI is not set, the backend automatically
  falls back to an in-memory database. Data resets on every restart.

- In production (NODE_ENV=production), if MONGODB_URI is missing or the
  Atlas connection fails at startup, the server exits immediately with a
  FATAL error instead of silently losing data.

- Make sure your Atlas cluster has your VPS IP address whitelisted under
  Network Access, or use 0.0.0.0/0 (allow all) for simplicity.

- Test your connection string before deploying:

      docker run --rm \
        -e MONGODB_URI=your_string \
        -e JWT_SECRET=any_value \
        -e NODE_ENV=production \
        halabunver:v1.0 \
        node -e "require('./src/config/db')()"


ENVIRONMENT VARIABLES SUMMARY
-------------------------------
MONGODB_URI      - (required) Your MongoDB Atlas connection string
JWT_SECRET       - (required) Your saved JWT secret — never change this after first run
JWT_EXPIRES_IN   - (optional) Token expiry duration, default: 7d
PORT             - (optional) Internal server port, default: 8000
NODE_ENV         - Set to "production" automatically by the Dockerfile


USEFUL DOCKER COMMANDS
-----------------------
View live logs:
    docker logs -f halabunver

View last 100 lines of logs:
    docker logs --tail 100 halabunver

Stop the container:
    docker stop halabunver

Start a stopped container (keeps env variables):
    docker start halabunver

Remove the container (needed before running a fresh one):
    docker rm -f halabunver

Check if the container is running:
    docker ps

Check resource usage (CPU, memory):
    docker stats halabunver

Health check (test the API is responding):
    curl http://localhost:8000/api/health

Open a shell inside the running container:
    docker exec -it halabunver sh


UPGRADING TO A NEW VERSION
----------------------------
1. Upload updated project files to the VPS.
2. Build the new image with a new tag:
       docker build --no-cache -t halabunver:v1.1 .
3. Stop and remove the old container:
       docker rm -f halabunver
4. Start a new container with the same env file and the new image:
       docker run -d --restart unless-stopped --env-file /etc/halabunver.env -p 8000:8000 --name halabunver halabunver:v1.1
