1. THE "LOW-COST" VALIDATION (WHY $6–$10 WORKS)
--------------------------------------------------------------------------------
* Media Offloading: By using YouTube for tutorials and Google Drive for PDFs, 
  you bypass the most expensive parts of web hosting: storage space and outbound 
  bandwidth. Your server only handles small JSON text strings.
* Database Efficiency: 100,000 user records and resource links typically occupy 
  less than 1 GB of storage. This fits perfectly within the free tiers of 
  modern cloud databases.
* Node.js Architecture: The non-blocking I/O of the MERN stack allows a 
  single-core entry-level VPS to handle thousands of "Read" requests 
  simultaneously without breaking a sweat.

3. RISK MANAGEMENT & SCALING "TRAPS"
--------------------------------------------------------------------------------
* The "Noisy Neighbor" Effect: Shared CPU power on a $5 VPS can be stolen by 
  other users on the same hardware.
  - Solution: Use Nginx caching so the server doesn't work for repeat visitors.
* Bandwidth Overage: 100k students downloading the React "Bundle" (code) can 
  cross the 1TB transfer limit of cheap plans.
  - Solution: Use a free CDN like Cloudflare to cache frontend files globally.
* Database Complexity: Growing data from the "Service Marketplace" and "Store."
  - Solution: Use the .lean() function in Mongoose queries to reduce RAM by 70%.

4. PROFESSIONAL IMPLEMENTATION CHECKLIST (THE "PRO" RULES)
--------------------------------------------------------------------------------
* Nginx Reverse Proxy: Essential for handling high concurrent traffic and security.
* JWT Authentication: Keeps the server "stateless" to save RAM on active sessions.
* Environment Optimization: Set NODE_ENV=production to boost speed and stop logs.
* PRO TIP: Use Nginx caching. This makes your $5 server perform like a $50 server.
